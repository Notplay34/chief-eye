# Smoke-check

Короткий финальный smoke-набор после изменений или деплоя.

## 1. Автоматическая локальная проверка

Полный релизный прогон одной командой:

```bash
bash scripts/release_smoke.sh
```

Скрипт создаёт тестовые заказы, платежи, строки касс, номера и складские операции только во временных pytest-БД. После завершения тестовые данные не остаются в рабочей базе, а локальные `__pycache__` / `.pytest_cache` очищаются.

Ручной эквивалент:

```bash
python3 -m pytest backend/tests -q
python3 -m compileall backend/app
node --check frontend/*.js frontend/form-page/*.js
```

Что покрывается:

- логин и `/auth/me`
- создание заказа и оплата
- касса и смены
- касса документов, списание госпошлин и деньги за номера
- склад
- аналитика
- access matrix по ролям: admin / manager / operator / plate operator
- сквозной сценарий: заказ, оплата, документы, склад, номера, промежуточная касса, касса номеров, аналитика

## 2. Ручная проверка ролей в браузере

Проверьте по одному входу на роль:

- `ROLE_ADMIN`
  - видит аналитику, админку и управление аккаунтами
  - открывает `analytics-docs.html` и `analytics-plates.html`
- `ROLE_MANAGER`
  - видит оба павильона, кассы и склад
  - не должен открывать аналитику
- `ROLE_OPERATOR`
  - видит форму и кассу павильона 1
  - не видит склад, номера и аналитику
- `ROLE_PLATE_OPERATOR`
  - видит номера, кассу номеров и склад
  - не видит форму и аналитику
  - не должен открывать или менять кассу павильона 1 (`/cash/rows`, выплаты за номера)
  - должен открывать кассу номеров и смены павильона 2

## 3. Базовые сценарии

Проверьте вручную:

1. Создать заказ без номеров.
2. Оплатить заказ и убедиться, что строка появилась в кассе.
3. Открыть страницу кассы документов и проверить текущий день.
4. Списать госпошлины за день и убедиться, что повторное списание не создаёт дубль.
5. Создать заказ с номерами.
6. Провести оплату, затем доплату за номера.
7. Перенести деньги за номера в промежуточную кассу.
8. Передать деньги из промежуточной кассы в кассу номеров.
9. Перевести заказ по статусам до готовности и проверить склад.
10. Проверить аналитику директора:
   - мой доход
   - оборот в кассе
   - источники дохода
   - проходные госпошлины
   - месяцы
   - сотрудники

## 4. После деплоя на сервер

Запуск:

```bash
bash deploy/check_stack.sh
```

Скрипт проверяет:

- `/health` напрямую и через nginx
- вход по `SUPERUSER_*`
- `/auth/me` напрямую и через nginx
- `/analytics/dashboard` через nginx
- отдачу основных статических страниц

Если smoke не проходит, сначала смотрите:

```bash
systemctl status eye_w
journalctl -u eye_w --no-pager -n 100
nginx -t
```
