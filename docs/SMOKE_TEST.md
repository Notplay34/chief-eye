# Smoke-check

Короткий финальный smoke-набор после изменений или деплоя.

## 1. Автоматическая локальная проверка

```bash
python3 -m pytest backend/tests -q
python3 -m compileall backend/app
node --check frontend/*.js frontend/form-page/*.js
```

Что покрывается:

- логин и `/auth/me`
- создание заказа и оплата
- касса и смены
- кассовый день и сверка фактической суммы
- склад
- аналитика
- access matrix по ролям: admin / manager / operator / plate operator

## 2. Ручная проверка ролей в браузере

Проверьте по одному входу на роль:

- `ROLE_ADMIN`
  - видит аналитику, админку и управление аккаунтами
  - открывает `analytics-docs.html` и `analytics-plates.html`
- `ROLE_MANAGER`
  - видит оба павильона, кассы и склад
  - не должен открывать аналитику
- `ROLE_OPERATOR`
  - видит форму и кассу павильона 1
  - не видит склад, номера и аналитику
- `ROLE_PLATE_OPERATOR`
  - видит номера, кассу номеров и склад
  - не видит форму и аналитику

## 3. Базовые сценарии

Проверьте вручную:

1. Создать заказ без номеров.
2. Оплатить заказ и убедиться, что строка появилась в кассе.
3. Открыть страницу кассы и проверить блок `Кассовый день`.
4. Ввести фактическую сумму и сверить день.
5. Убедиться, что при несовпадении отображается расхождение.
6. Создать заказ с номерами.
7. Провести оплату, затем доплату за номера.
8. Перевести заказ по статусам до готовности.
9. Проверить, что склад резервируется и списывается корректно.
10. Проверить аналитику директора:
   - доход
   - оборот
   - месяцы
   - кварталы
   - сотрудники
   - статусы

## 4. После деплоя на сервер

Запуск:

```bash
bash deploy/check_stack.sh
```

Скрипт проверяет:

- `/health` напрямую и через nginx
- вход по `SUPERUSER_*`
- `/auth/me` напрямую и через nginx
- `/analytics/dashboard` через nginx
- отдачу основных статических страниц

Если smoke не проходит, сначала смотрите:

```bash
systemctl status eye_w
journalctl -u eye_w --no-pager -n 100
nginx -t
```
