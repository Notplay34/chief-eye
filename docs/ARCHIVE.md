# Archive Guide

Этот файл описывает документы, которые сохранены в репозитории как история,
а не как основной вход в проект.

## Не использовать как главную документацию

- `DEVELOPMENT_PLAN.md` — ранний план разработки, отражает стартовый замысел, а не текущее состояние.
- `RELEASE_PLAN.md` — исторический релизный план и чеклисты прошлых версий.
- `DevelopmentDiary.md` — дневник промежуточных экспериментов и удалённых направлений.
- `PROJECT_REVIEW.md` — аудит проекта на конкретный момент времени.
- `DEVELOPMENT_LOG.md` — подробная хронология изменений, полезна как журнал, но не как стартовая документация.
- `docs/ARCHITECTURE_RBAC_AND_MAIN_PAGE.md` — архитектурная заметка о целевом направлении, не полная спецификация фактической реализации.
- `frontend/FRONTEND_REFACTOR.md` — заметка о фронтенд-рефакторинге, полезна только при работе над UI-слоем.

## Что читать вместо этого

Если задача прикладная, использовать в таком порядке:

1. `README.md`
2. `PROJECT_CONTEXT.md`
3. `INSTALL.md`
4. `USER_GUIDE.md`

Для серверной настройки дополнительно:

1. `DEPLOYMENT.md`
2. `docs/MIGRATIONS.md`
