# Repository State

Этот файл описывает только текущий поддерживаемый контур проекта.

## Рабочие каталоги

- `backend/` — FastAPI backend, бизнес-логика, модели, API и тесты.
- `frontend/` — рабочие страницы, общие скрипты и стили.
- `templates/` — `.docx` шаблоны для генерации документов.
- `deploy/` — nginx-конфиг, серверные скрипты и smoke-check.
- `docs/` — рабочая документация по проекту.

## Основные документы

- `README.md` — обзор проекта и текущих возможностей.
- `PROJECT_CONTEXT.md` — бизнес-логика, роли и границы системы.
- `INSTALL.md` — локальный запуск.
- `USER_GUIDE.md` — сценарии для пользователей.
- `DEPLOYMENT.md` — выкладка на сервер.
- `docs/MIGRATIONS.md` — история изменения схемы БД.
- `docs/SMOKE_TEST.md` — финальная проверка после изменений и деплоя.

## Что читать в первую очередь

1. `README.md`
2. `PROJECT_CONTEXT.md`
3. `INSTALL.md`
4. `USER_GUIDE.md`
5. `DEPLOYMENT.md`

## Что сознательно отсутствует

В репозитории больше не хранятся старые планы, дневники и аудитные заметки как отдельный слой документации. Оставлены только документы, которые должны описывать текущее состояние проекта.
