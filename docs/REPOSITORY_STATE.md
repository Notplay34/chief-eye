# Repository State

Этот файл фиксирует текущее состояние репозитория и помогает быстро понять,
какие части проекта являются рабочим контуром, а какие относятся к истории
разработки, аудиту или планированию.

## Живой контур

- `backend/` — основной backend проекта: FastAPI, SQLAlchemy, бизнес-логика, API, тесты.
- `frontend/` — рабочие интерфейсы операторов, кассы, склада, аккаунтов и админских страниц.
- `templates/` — `.docx`-шаблоны для генерации документов.
- `deploy/` — инфраструктурные файлы для nginx и серверного развёртывания.
- `docker-compose.yml` — локальный PostgreSQL для запуска и тестового окружения.

## Основные документы

- `README.md` — входная точка по проекту и его назначению.
- `PROJECT_CONTEXT.md` — бизнес-контекст и целевая логика системы.
- `INSTALL.md` — установка и первый запуск.
- `USER_GUIDE.md` — пользовательские сценарии.
- `DEPLOYMENT.md` — серверное развёртывание.
- `docs/MIGRATIONS.md` — журнал изменения схемы БД.

## Исторические и служебные документы

Эти файлы полезны как история решений, но не должны восприниматься как
текущее состояние проекта по умолчанию. Сводка по ним также вынесена в
`docs/ARCHIVE.md`:

- `DEVELOPMENT_LOG.md` — хронология изменений и рабочих итераций.
- `DEVELOPMENT_PLAN.md` — ранний план разработки; частично устарел.
- `DevelopmentDiary.md` — исторические заметки по экспериментам и промежуточным идеям.
- `PROJECT_REVIEW.md` — аудит состояния проекта на момент проверки; не спецификация.
- `RELEASE_PLAN.md` — релизный план версии 1.0 / 1.1, использовать как исторический roadmap.
- `docs/ARCHITECTURE_RBAC_AND_MAIN_PAGE.md` — целевая архитектурная заметка, не полное описание фактической реализации.
- `frontend/FRONTEND_REFACTOR.md` — заметка о фронтенд-рефакторинге, не пользовательская документация.

## Принцип навигации по репозиторию

Если нужно понять проект с нуля:

1. Сначала читать `README.md`.
2. Затем `PROJECT_CONTEXT.md`, чтобы понять бизнес-логику.
3. Для запуска — `INSTALL.md`.
4. Для пользовательских сценариев — `USER_GUIDE.md`.
5. Для продакшн-развёртывания — `DEPLOYMENT.md`.
6. Для деталей реализации — уже после этого смотреть код в `backend/` и `frontend/`.
